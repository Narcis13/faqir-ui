Task Manager v2

Declarative REST binding — l-source:tasks.optimistic on one attribute replaces an entire data layer.

Zero imperative fetch code

Everything below — loading, error, create, update, delete, optimistic rollback and polling — is driven by a single attribute on the root element. No fetch(), no useEffect, no store. 

<div l-data="{ ... }"
     l-source:tasks.optimistic="/api/tasks">

That one line injects four reactive members into scope: tasks (the array), tasksLoading (boolean), tasksError (string or null), and $tasks — the controller exposing create(), update(), remove(), refresh() and startPolling(). Press Tab to walk the whole board by keyboard.
