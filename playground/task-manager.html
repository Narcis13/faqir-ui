What is l-source?

l-source is a declarative data-binding directive. Adding l-source:tasks="/api/tasks" to an l-data element auto-fetches data and injects tasks (array), tasksLoading (boolean), tasksError (string|null), and $tasks (controller with load(), create(), update(), remove(), refresh(), startPolling(), stopPolling()). This page uses the .optimistic modifier for instant UI updates and a second source (l-source:categories) for category data.

This section uses a second l-source:categories="/api/categories" on the same l-data element, demonstrating multiple sources. It injects categories, categoriesLoading, categoriesError, and $categories into the scope.

l-source Syntax Reference

l-source:name="endpoint" — basic usage
l-source:name.lazy="endpoint" — don't auto-load on init
l-source:name.poll.5000="endpoint" — auto-poll every 5s
l-source:name.optimistic="endpoint" — optimistic UI updates
l-source:name.key.uuid="endpoint" — custom ID field name
